Computers connected to Web are called client and servers.

  • Client are typical Web user's Internet-connected devives (for example, your computer connnected to your wifi, your phone connected to your mobike network) and Web accessing software available on those devices (usually a web browser like Firefox or Chrome).
  • Servers are computers that store webpages, sites and apps. When client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine to be
    displayed in the user's web browser.

So What Actually Happens?

  1. The browser goes to DNS server and finds the real address of the server that the website lives on.
  2. The browser sends an HTTP request message to the server asking it to send a copy of the website to the client.
  3. If server approves the client's request, the server sends the client "200 OK" message, which means "Permission granted! Here you go!" and then start sending the website's file to the browser as a small series of chunk.
  4. The browser assembles the small chunks into complete website and displays it to you!

The Big 3 - HTML, CSS, JavaScript
